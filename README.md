# Order Flow App Frontend（React / TypeScript）

Order Flow App のフロントエンドリポジトリです。

## 概要

Order Flow App は、店舗で受け付けた注文を管理するための注文管理アプリです。

本リポジトリでは、注文作成フロー、注文管理画面、商品管理画面、店舗管理画面などのUIを React / TypeScript で実装しています。

## 作成背景

現職では、商品の注文受付にFAXを使用しており、店舗と製造側の連携に課題がありました。

注文不可の商品や、製造が間に合わない日時の注文を受け付けてしまう可能性があるため、注文内容・受け渡し方法・希望日時・ステータスを画面上で確認・管理できるアプリとして作成しました。

## 画面構成

### 注文フロー

店舗ユーザーが注文を作成するための画面です。

- 希望日時の入力
- 商品選択
- カート確認
- 顧客情報の入力
- 受け渡し方法の選択
- 注文内容の確認

### 注文管理

作成済み注文の一覧・詳細確認・更新を行う画面です。

- 注文一覧
- 注文詳細
- 顧客情報の更新
- 商品内容の更新
- 受け渡し方法の変更
- 希望日時の変更
- 注文ステータスの変更

### 商品管理

admin が注文フローで使用する商品を管理する画面です。

- 商品の新規作成
- 商品情報の更新
- 表示 / 非表示の切り替え

### 店舗管理

admin が注文受付店舗を管理する画面です。

- 店舗の新規作成
- 店舗情報の更新
- 表示 / 非表示の切り替え

## 注文フロー

注文作成は、店舗での受付業務を想定し、以下の順番で進める構成にしています。

1. 希望日時の入力
2. 商品選択
3. カート確認
4. 顧客情報の入力
5. 受け渡し方法の選択
6. 注文内容の確認

最初に希望日時を入力することで、製造が間に合わない日時の注文を受け付けないようにしています。

また、商品選択後にカート確認画面を設けることで、注文確定前に商品内容・数量を確認できるようにしています。

## 権限制御

ログイン中ユーザーの role に応じて、アクセスできる画面を制御しています。

- admin：商品管理、店舗管理、注文管理にアクセス可能
- store_user：注文作成、自店舗の注文管理にアクセス可能

フロントエンド側では、React Router のルーティングとログインユーザー情報をもとに画面アクセスを制御しています。

ただし、不正操作の最終的な制御はバックエンド側の Laravel Policy でも行い、フロントエンドの表示制御だけに依存しない設計にしています。

## API連携・状態管理

Laravel API との通信には、主に Axios を使用しています。

認証はバックエンドの Laravel Sanctum と連携し、ログイン状態をもとに画面制御を行っています。

ログイン中ユーザーの取得には TanStack Query を使用し、ログイン状態や role に応じて画面表示・アクセス制御を切り替えています。

## バリデーション・エラーハンドリング

フォーム入力時には、フロントエンド側で必要な入力制御を行い、送信後にバックエンド側のバリデーションエラーが返った場合は、エラーメッセージを画面上に表示するようにしています。

Laravel API から返る 422 エラーについては、レスポンス内容からメッセージを取得し、toast でユーザーに通知しています。

これにより、不正な入力や業務ルールに反する操作があった場合でも、ユーザーが原因を把握しやすいようにしています。

## セットアップ（ローカル環境）

事前にバックエンド（Laravel API）を起動しておきます。

```bash
npm install
npm run dev
```

開発サーバー起動後、以下のURLでアクセスできます。

```txt
http://localhost:5173
```
